并发雪花算法

并发雪花算法

网上已经有很多雪花算法（Snowflake）的实现教程，但是这些教程大多没有考虑单机的并发带来的竞态条件，或者直接给整个方法加 synchronzied，使方法的运行串行化，大大降低了并发性能。 本文试图实现一个高并发性能的雪花算法。

雪花算法所需的基本常量如下：

常量名 含义 类型 值
TIMESTAMP_BITS 时间戳位数 int 41
WORKER_ID_BITS Woker ID 位数 int 10
SEQUENCE_BITS 序列号位数 int 12
EPOCH 初始时间 long
CLOCK_BACKWARD_TOLERANCE 时钟回拨容忍度 long

基于 CAS 的雪花算法

雪花算法中存在竞态条件的共享资源主要是两个： 上次生成 id 时的时间戳 lastTimestamp 和序列号 sequence。 因此一个显而易见的想法，就是构建状态对象来保存这两个属性，并利用原子引用 AtomicReference 实现状态变更操作的原子化。

基于 CAS 的雪花算法的主要流程如下：

  1. 其核心是对 State 对象的原子引用 globalState，它描述了系统上次生成 id 时的时间戳 lastTimestamp 和序列号 sequence。
  2. 每次生成新 id 时，generate 方法都会获取 globalState 引用的状态对象，将当前时间戳和上次时间戳进行比较，根据不同的情形选择不同的行为。
  3. 如果 currentTimeStamp < lastTimestamp，说明发生了时间回拨。 如果回拨在容忍值范围内，则直接将当前时间戳调整为上次时间戳，然后继续方法； 如果超过容忍值则报错。
  4. 如果 currentTimeStamp == lastTimestamp，说明仍处于同一微秒，此时将 sequence 加 1。 如果 sequence 已经超过了上限，则等待进入下一微秒。
  5. 如果 currentTimeStamp > lastTimestamp，说明已进入了新的微秒，此时重置 sequence 为 0。
  6. 根据 currentTimeStamp 和 sequence 生成新的状态对象，以 CAS 的方式赋值给 globalState。 如果有两个线程同时要更新 globalState，由于更新操作是 CAS，只会有一个线程成功，另一个线程重新上述流程，直至成功。
